★Gitの操作方法★

★Gitの基本操作★

①ローカルリポジトリの作成
  • mkdir [フォルダ名]
  • cd [フォルダ名]
  • git init
    • git initコマンドはGitリポジトリを新たに作成するコマンド
    ②ファイルの作成・変更・削除をgitインデックスに追加する(git add)
  • git add [ファイル名］
    • インデックスとはリポジトリにコミットする準備をするために変更内容を一時的に保存する場所のことです
    ③変更内容をローカルリポジトリにコミットする(git commit)
  • git commit -m “コメント”
    • 変更内容をコメントに残してコミットする
  • git status
    • ファイルが追加されているか確認
  • git remote add origin [リモートリポジトリのアドレス]
    • リモートリポジトリに反映される前にリモートリポジトリの情報を追加する
    ④ローカルリポジトリをプッシュしてリモートリポジトリに反映させる(git push)
  • git push origin master
    • この時にユーザ名とパスワードを聞かれる

    ★ブランチの作成★

    現在のブランチ一覧を確認
  • git branch
    • ブランチを作成
  • git branch [ブランチ名]
    • ブランチの移動
  • git branch checkout [ブランチ名]
    • ※ブランチの作成と移動
  • git branch -b [ブランチ名]
    • リモートリポジトリに反映
  • git push origin [ブランチ名]
  • ※git push -f origin [ブランチ名]
    • 強制的にプッシュする

    ★ブランチからプル★

    他の開発者がブランチで作業を行う場合、 作成したリポジトリにチェックアウトする
  • git checkout [ブランチ名]
    • リモートブランチのコードを取得する
  • git pull
    • ファイルの確認
  • ls

    • ★ブランチのマージ★

    作業中のブランチをメインのブランチに切り替える
  • git checkout [メインのブランチ名]
    • 作業用のブランチをメインのブランチにマージする
  • git merge [作業用のブランチ名]
    • リモートリポジトリに反映させる
  • git push origin [メインのブランチ名]

    • ★ブランチの削除★

    使わなくなったブランチの削除
  • git branch -d [削除するブランチ名]
    • 結果の確認
  • git status

    • ★よく使うコマンド★

    リポジトリの状態を確認する
  • git status
    • ファイルやディレクトリをインデックスに追加する
  • git add [ファイル名]
    • インデックスに追加されたファイルやフォルダの変更をリポジトリに書き込む
  • git commit [オプション]　”コメント”
    • オプション
    -a 変更されたファイルを検出してインデックスに追加する動作も同時に行います -m ダブルクォーテーションで囲ったメッセージを指定する

    ★git branch ブランチに対して各種操作を行う

    ブランチの作成
  • git branch [ブランチ名]
    • ブランチの一覧表示
  • git branch
    • 指定したブランチの削除
  • git branch -d [ブランチ名]
    • ローカルリポジトリのブランチを切り替える
  • git checkout [ブランチ名]
    • ローカルリポジトリのコミット履歴を閲覧
  • git log [オプション]
    • オプション
    -n 履歴の表示数を指定できる
    リポジトリのファイルの内容から検索する
  • git grep “検索単語”
    • 既存のリモートリポジトリをローカルに落とす
  • git clone [既存のリモートリポジトリのURL]

    • ★git remote リモートリポジトリを操作する

    リモートリポジトリの名称一覧を表示
  • git remote
    • リモートリポジトリの詳細一覧を表示
  • git remote -v
    • リモートリポジトリを追加
  • git remote add [リポジトリ名] [リモートリポジトリのURL]
    • リモートリポジトリを削除
  • git remote rm [リポジトリ名]
    • ローカルリポジトリのコミットを取り消す
  • git reset
    • 現在のブランチに対して他のブランチで行った変更を取り込む
  • git merge
    • 作業していたブランチをメインのブランチへ変更を取り込む
  • git merge [作業していたブランチ]
    • 例）git checkout [メインのブランチ]
    リモートリポジトリの変更を取り込む
  • git pull
    • 例）git checkout [ローカルリポジトリのブランチ]
    ローカルリポジトリのブランチにリモートリポジトリのブランチを取り込む
  • git pull [リモートリポジトリのブランチ]
